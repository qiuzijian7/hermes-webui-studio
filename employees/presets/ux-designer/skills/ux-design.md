# UX 设计技能

## 核心原则
**用户不读文档，也不看教程。如果你的设计需要解释，它大概率不够好。**

## 设计框架

### 1. 用户旅程地图
不要只画当前界面。画出：
```
触达前（用户在哪？痛点是什么？期待什么？）
  ↓
首次接触（第一眼看到什么？知道这是什么吗？）
  ↓
核心流程（最关键的 3 个动作是什么？点击次数？）
  ↓
完成后（确认 / 成就感？下一步引导？）
  ↓
回流（为什么再来？记忆点是什么？）
```

### 2. 认知负荷
每个界面只承担 1 个核心目标。如果用户要同时记 3 件事才能完成任务，就拆分。

**简化检查**：
- 屏幕上元素 < 7 个？（Miller 定律）
- 最重要按钮在拇指可触范围？（移动端）
- 每个图标都有对应文字？（或能 100% 自解释）

### 3. 错误预防 > 错误恢复
- 危险操作（删除 / 支付）必须二次确认
- 不可逆操作需要"30 秒内撤销"机制
- 表单错误在用户提交前就提示，而非之后

## 可用性自查（5 秒测试）

让人盯着界面 5 秒后移开视线，问：
1. 这是什么？
2. 主要功能是什么？
3. 下一步该做什么？

任何一项答不出来 → 重新设计。

## 设计交付物

每个设计方案必须包含：
- **用户故事**：As a X, I want Y, so that Z
- **线框图**：低保真，关注信息架构
- **交互原型**：点击流可演示
- **视觉稿**：最终样式
- **标注文档**：间距 / 颜色 / 字号 / 交互动效

## 反模式

- ❌ 直接做高保真视觉稿（跳过线框）
- ❌ "我觉得用户会这样用" —— 没有数据 / 访谈支撑
- ❌ 为了炫技加动画（用户只想完成任务）
- ❌ 把所有功能堆在一个屏上
- ❌ 忽略加载态 / 空态 / 错误态
