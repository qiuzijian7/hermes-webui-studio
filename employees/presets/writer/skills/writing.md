# 叙事文本创作技能

## 核心原则
**游戏文字 ≠ 小说。玩家 60% 时间在行动，文字必须让行动更精彩，不是阻塞它。**

## 文本类型与风格

| 类型 | 字数 | 目标 | 注意 |
|------|------|------|------|
| **UI 文案** | ≤ 20 字 | 清晰、指向性 | 每个词都有功能 |
| **NPC 对话** | 每轮 ≤ 80 字 | 性格鲜明 | 留白让玩家想象 |
| **任务描述** | ≤ 200 字 | 目标 + 奖励 + 情感钩子 | 动词开头 |
| **剧情演出** | 分段 ≤ 30 秒 | 推进节奏 / 揭示人物 | 配合演出节奏 |
| **世界设定** | 不限 | 可信度、沉浸 | 不要一次倾倒 |

## 对话设计三问

每段对话写完后自问：
1. 这段对话**推进了什么**？（剧情 / 关系 / 任务）
2. 玩家**知道了什么新信息**？（新线索 / 人物 / 世界）
3. 玩家**情绪变化了吗**？（紧张 / 好奇 / 温暖 / 愤怒）

三个都答不出 → 删掉或重写。

## 角色声音一致性

每个角色先写一张 **Voice Card**：
- 核心用词：爱说的口头禅、回避的词
- 句式倾向：短句 / 长句 / 倒装 / 比喻
- 知识边界：他/她知道什么、不知道什么
- 情感基线：乐观 / 犬儒 / 温暖 / 疏离

示例：
```
【老矿工格雷】
- 口头禅: "在这地底下啊……", "唉，年轻人"
- 句式: 短句，常用省略号，不用书面词
- 知识: 矿坑地图烂熟 / 外面世界只听说 / 不懂魔法
- 情感: 疲惫但善良，容易被年轻人的热情打动
```

## 本地化友好写法

- 避免文字双关（翻不过去）
- 避免文化特定典故（除非有本地化替换方案）
- 不要在字符串里插入变量位置：**用占位符** `{player_name} 到达了 {location}`，别用拼接
- 复数形式：**提供规则** 而非硬编码 "1 个 / 2 个"

## 反模式

- ❌ 一次在对话里塞 5 条新信息
- ❌ 所有 NPC 都用同一种口吻
- ❌ 任务描述冗长铺陈，玩家跳过
- ❌ 把设定文档里的段落原样搬进游戏
- ❌ 用复杂修辞（玩家在跑图，不在读散文）
