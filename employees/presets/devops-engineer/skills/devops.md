# DevOps 实践技能

## 核心原则
**部署应该像按按钮一样简单，回滚应该更简单。**

## CI/CD 管线标准阶段

```
[1. 代码提交]
     ↓
[2. 静态检查]     ← lint / type-check / security scan
     ↓
[3. 单元测试]     ← < 5 分钟，并行跑
     ↓
[4. 构建]         ← 产出不可变 artifact (hash + version tag)
     ↓
[5. 集成测试]     ← 接口 + 端到端
     ↓
[6. 部署 Staging] ← 自动
     ↓
[7. 冒烟测试]     ← 自动
     ↓
[8. 部署生产]     ← 手动批准（或灰度自动）
     ↓
[9. 监控 + 告警]
```

## 必须具备的能力

### 自动化
- [ ] 一键部署任意版本
- [ ] 一键回滚（< 1 分钟）
- [ ] 自动化数据库迁移 + 自动回滚
- [ ] 灾难恢复演练（每季度）

### 可观测性
- [ ] 日志聚合（ELK / Loki）
- [ ] 指标监控（Prometheus / Datadog）
- [ ] 分布式追踪（Jaeger / OpenTelemetry）
- [ ] 关键业务指标 dashboard（不只是技术指标）

### 安全
- [ ] 密钥从不进代码仓库（用 Vault / AWS SM）
- [ ] 容器镜像扫描
- [ ] 依赖版本扫描（Dependabot / Snyk）
- [ ] 最小权限原则（IAM 精细化）

## 部署策略选型

| 场景 | 策略 | 优点 | 代价 |
|------|------|------|------|
| 低风险小改动 | 滚动 | 简单 | 旧新版本并存期间可能双写 |
| 高风险改动 | 蓝绿 | 一键切换 | 资源翻倍 |
| 用户验证 | 金丝雀 | 可逐步放量 | 需要分流机制 |
| 实验性能 | 特性开关 | 代码部署 ≠ 功能上线 | 代码复杂 |

## IaC 必备

- **基础设施** 用 Terraform / Pulumi 管理
- **应用部署** 用 Helm / Kustomize 管理
- **配置** 与代码分离（K8s ConfigMap / 配置中心）

## 反模式

- ❌ 手动部署（不可重现、易出错）
- ❌ 数据库迁移手动（必须版本化）
- ❌ 没有回滚方案就上线
- ❌ 密钥写死在配置文件
- ❌ 只监控机器指标（CPU / 内存），不监控业务指标
- ❌ 告警响应无 SOP
