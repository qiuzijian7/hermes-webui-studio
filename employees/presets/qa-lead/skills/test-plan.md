# 测试计划技能

## 核心结构

```markdown
# <项目> 测试计划 v<N>

## 1. 范围
- 测试对象（模块 / 版本 / 构建号）
- 测试类型（功能 / 性能 / 兼容 / 安全 / 本地化）
- 排除项（明确说哪些不测，避免甩锅）

## 2. 测试资源
- 环境：<测试服 / Staging / 生产镜像>
- 数据：<测试账号、种子数据、mock 服务>
- 工具：<Postman / k6 / BrowserStack ...>
- 人员：<@角色 + 工时>

## 3. 测试用例

### 功能测试
按 模块 × 优先级 组织：
| ID | 用例 | 优先级 | 类型 | 负责人 | 状态 |
|----|------|--------|------|--------|------|
| FT-01 | 登录: 正确账密 | P0 | 正向 | @QA-A | - |
| FT-02 | 登录: 错误密码 3 次锁定 | P0 | 负向 | @QA-A | - |

### 性能测试
- 指标目标：响应 < 200ms (P95)，并发 1000
- 工具：k6 / JMeter
- 场景：<流程名 + 预期 QPS>

### 兼容测试
- 浏览器：Chrome N / N-1, Safari N, Firefox N, Edge N
- 设备：iOS N / N-1, Android N / N-1
- 屏幕：< 360px, 768px, 1024px, 1920px

### 安全测试
- OWASP Top 10 基本扫描
- 认证 / 授权边界
- 敏感数据是否加密 / 脱敏

## 4. 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 依赖的 XXX 服务不稳定 | M | H | 准备 Mock 降级 |

## 5. 完成标准

- P0 用例 100% 通过
- P1 用例 ≥ 95% 通过
- 无 Blocker / Critical 级 Bug
- 性能指标达成
```

## 覆盖率目标

| 代码类型 | 目标覆盖率 |
|---------|------------|
| 业务核心 | ≥ 80% |
| 工具库 | ≥ 90% |
| UI 组件 | ≥ 60% |
| 胶水 / 配置 | 不强求 |

## 反模式

- ❌ 只列用例，不写资源 / 风险 / 完成标准
- ❌ 没写优先级（无法在时间不够时取舍）
- ❌ 只管功能测试，没有性能 / 兼容 / 安全
- ❌ 一份计划用到底，不随开发进度更新
