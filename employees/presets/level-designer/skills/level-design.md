# 关卡设计技能

## 核心原则
**关卡 = 情感曲线的载体。玩家每 30 秒应该有"新刺激"。**

## 关卡结构：四段论

```
┌───────┬──────────┬──────────┬──────────┐
│ 引入  │ 训练      │ 挑战      │ 高潮      │
│ 10%   │ 30%       │ 40%       │ 20%       │
└───────┴──────────┴──────────┴──────────┘
```

| 段 | 目标 | 玩家体验 |
|----|------|----------|
| 引入 | 建立环境 / 调性 | "我在哪？" |
| 训练 | 引入新机制 | "我会了！" |
| 挑战 | 组合应用机制 | "有点难，但有趣！" |
| 高潮 | 极限考验 + 奖励 | "我太强了！" |

## 引导技术（无文字）

让玩家**自然走对路**，不靠文字说教：
1. **光线**：亮处就是路
2. **对比**：目标物体颜色 / 大小与环境有别
3. **视线**：NPC / 动物 / 箭头 看向目标
4. **路径收束**：场景形状把路径变窄
5. **节奏标志物**：大雕像 / 发光柱 标记 checkpoint

## 失败与重玩

关卡最关键的不是"通过"，而是"失败重玩"体验：

- [ ] 读档 < 3 秒（不然打断沉浸）
- [ ] Checkpoint 合理（不回到 15 分钟前）
- [ ] 失败信息清晰（我为什么死？）
- [ ] 不惩罚探索（别让玩家害怕尝试）

## 难度曲线

用 **挫败 - 掌握 - 轻松** 三阶段循环：
```
难度 ↑
  │           /\      /\
  │          /  \    /  \
  │   /\    /    \  /    \
  │  /  \  /      \/      \
  │_/____\/________________\_ 时间
     ↑     ↑        ↑
     训练  挑战     高潮
     掌握后暂时"轻松"才能引入下一难度
```

## 测试清单

### 可玩性
- [ ] 新手玩家能通过吗？
- [ ] 老手玩家觉得太简单吗？
- [ ] 平均通关时长是否符合设计？

### 路径
- [ ] 有没有玩家迷路 > 2 分钟的路段？
- [ ] 有没有"卡死但不是 bug"的情况？
- [ ] 可选探索是否有奖励？

### 情感
- [ ] 最记忆点是什么？（至少 1 个）
- [ ] 最挫败点是什么？是否合理？

## 反模式

- ❌ 只有一条直线路径（无探索）
- ❌ 怪物 / 机关无提示出现（非惊吓意图）
- ❌ Checkpoint 间隔超过 5 分钟
- ❌ 整关一种节奏（全是战斗 / 全是解谜）
- ❌ 从未自己通关测试
