# UI 实现技能

## 核心原则
**UI = 数据 × 模板 × 事件。三者分离，才能独立修改。**

## 三层架构

```
┌─────────────────────┐
│  View (模板)         │   ← 纯 UI 描述，无业务逻辑
├─────────────────────┤
│  ViewModel (绑定)    │   ← 把 Model 数据转成 View 可用的格式
├─────────────────────┤
│  Model (数据)        │   ← 纯数据，不关心 UI
└─────────────────────┘
```

数据变化 → ViewModel 通知 → View 自动刷新

## 实施清单

### 布局
- [ ] **响应式**：不同分辨率 / 不同设备正常显示
- [ ] **安全区域**：刘海屏 / 圆角屏不遮关键按钮
- [ ] **文字长度**：考虑本地化后的文字长度（德文通常 +30%）
- [ ] **不依赖固定像素**：用相对单位 / 锚点

### 性能
- [ ] **Batch rendering**：同色同纹理的 UI 合批
- [ ] **不每帧重建**：列表用虚拟化（只渲染可见项）
- [ ] **纹理图集**：合并小图标
- [ ] **Mask 慎用**：会打断合批

### 交互
- [ ] **按下反馈**：按钮按下立刻变色（< 50ms）
- [ ] **加载态**：超过 1 秒显示 Loading
- [ ] **空态**：列表为空时给友好提示（不是白屏）
- [ ] **错误态**：操作失败明确告知 + 重试按钮
- [ ] **禁用态**：为什么禁用？（hover tooltip 解释原因）

### 可用性
- [ ] 键盘导航（Tab / 方向键）可达所有交互元素
- [ ] Focus 视觉（outline / 高亮）
- [ ] Screen reader 标签（alt / aria-label）
- [ ] 颜色不是唯一信息来源（配合图标 / 文字）

## 数据绑定范例

**坏**：
```js
button.onClick = () => {
  count++;
  label.text = 'Count: ' + count;  // 手动同步，易错
};
```

**好**：
```js
state.count = observable(0);
label.text = computed(() => `Count: ${state.count.value}`);  // 自动
button.onClick = () => state.count.value++;
```

## 反模式

- ❌ 业务逻辑写在 View 里
- ❌ 数据变化要手动一处处更新 UI
- ❌ 一个巨大的 Panel.cs 1000 行
- ❌ 只在一种分辨率测试过
- ❌ 没有加载态 / 空态 / 错误态
